# Machine State Layer

Purpose:

- Define the complete machine state at one cycle time
- Use `cycle_time.py` for beam parameters
- Store all GUI/backend settings in one Python object
- Write one timestamped MAD-X-readable override/log file
- Keep the MAD-X lattice files unchanged

---

# Files

```text
machine_state.py
machine_state_defaults.py
machine_state_writer.py
tune_control.py
correctors.py
```

---

Main dataclass:

```text
hd_corrector_kicks_rad

hd_corrector_currents_A
vd_corrector_currents_A
```

Main methods:

```python
MachineState.from_defaults()

MachineState.calculate_trim_quad_strengths()

MachineState.update_harmonics()

MachineState.update_correctors()

MachineState.summary_dict()
```

Responsibilities:

```text
- Hold all machine settings for one time point
- Use beam state from cycle_time.py
- Calculate kqtd/kqtf if required
- Store both corrector kicks and currents
- Store harmonic tune settings
- Store applied MAD-X error table
- Act as the GUI/backend state object
```

Functions:



# machine_state_writer.py

Purpose:

Functions:

Output example:

```madx
qf_scale := ...;
```

# Initial Workflow

```python
from cycle_time import RCSRamp
from machine_state import MachineState
from machine_state_writer import write_machine_state_file

ramp = RCSRamp(top_energy_MeV=800.0)

beam_state = ramp.state_at(3.0)

machine_state = MachineState.from_defaults(
    beam_state=beam_state,
    main_magnet_mode="srm_bare",
    requested_qx=4.31,
    requested_qy=3.76,
    tune_method="di_wright",
)

write_machine_state_file(
    machine_state,
    output_dir="./machine_states",
)
```

Output:

```text
machine_states/
    machine_state_20260514_153012_t3p000ms.strength
```

This file acts as:

```text
- MAD-X override file
- GUI snapshot
- logged machine-state record
```
